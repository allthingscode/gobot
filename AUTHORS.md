## Featured Work: Dev Factory + Gobot

**Dev Factory** is a private, production-grade agentic development ecosystem I designed and built from the ground up. 
It demonstrates autonomous software engineering at a level most teams only theorize about:

- **Architecture**: 5-specialist recursive improvement system (Architect, Coder, Reviewer, Tester, Documentation agents) with structured handoff protocols and human-in-the-loop gates
- **Scale**: End-to-end feature delivery — from backlog ticket to committed, tested, documented code — driven primarily by agents
- **Reliability**: Built on formal task protocols, spec-driven handoffs, and strict output contracts between agents to prevent context drift and hallucination compounding
- **Tooling**: Agents are driven by Claude Code and Gemini CLI — industry-standard AI coding tools orchestrated through Dev Factory's protocol layer, not proprietary lock-in

[**Gobot**](https://github.com/allthingscode/gobot) is the live evaluation target — a self-hosted Go agent platform (Telegram-based, Google Workspace integration, Windows-native) that Dev Factory continuously develops, tests, and ships to. 
It serves as a real-world proving ground: if Dev Factory can reliably deliver production-quality features to Gobot autonomously, the system works.

## Experience

**Senior Agentic Software Engineer** · Pinger · Dallas, TX

Led the development and adoption of agentic AI tooling and workflows across the engineering team, with a dual focus on shipping velocity and software quality. 
Served as the internal lead on integrating AI coding tools into real-world team workflows — not as an experiment, but as a sustained operational improvement.

Core to this work is a philosophy: agentic AI tools are a force multiplier, but only in the hands of engineers who deeply understand both what AI does well and where it falls apart. The leverage is enormous when applied correctly. 
Applied without that judgment — to engineers who don't understand how LLMs reason, where they hallucinate, or how context degrades — AI tooling actively harms codebases: inflating complexity, eroding quality, and creating technical debt that compounds quickly and quietly.

My role was to bridge that gap: designing the workflows, guardrails, and team practices that let experienced engineers extract maximum value from agentic tools while preventing the failure modes that plague less-disciplined adoptions.

Central to that is knowing where AI belongs and where it doesn't. Agentic tools excel at reasoning, synthesis, and navigating ambiguity — but they are not a replacement for deterministic code and scripts where precision, reliability, and predictability are required. The highest-leverage systems are hybrid: AI handles the judgment-heavy work, deterministic logic handles everything where correctness must be guaranteed. Getting that boundary right is an art, and getting it wrong in either direction is costly.
